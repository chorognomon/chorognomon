# Threads

Stubs for investigations we want to come back to. Each thread is a short markdown file with:

- **Question** — what we want to figure out
- **What we know so far** — the context that surfaced the thread, citations, prior thinking
- **Why it matters** — what depends on resolving this
- **What would resolve it** — concrete moves that would close or advance the thread

Threads are deliberately *not* part of the project's analytical vocabulary (Niche, Campaign, etc.) — they're meta-work about the design of Chorognomon itself, kept separate so the noun "niche" can stay reserved for things the system reasons about.

When a thread ripens into actual spec work, move its conclusions into `spec/` and either close the thread file or leave it as a pointer.
